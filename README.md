# 1file-dualview-sat
A lightweight, standalone HTML5 Canvas app that visualizes satellite orbits in both Skyplot and Footprint views via a single HTML file.

## ほしかったのでつくってみました
 - 地上視点(skyplot)からの衛星と、衛星からの可視域(footprint)をブラウザで同時表示するツールです。
 - 前後3時間分をスライダで動かして可視化できます。
 - 単一のhtmlファイルのみで動作します。
 - TLEと現在位置はマウスでコピペできます。コピー元のリンクもページ内に埋め込んであります。

## README

本プロジェクトは Gemini Flash 3.5との対話を通じ、段階を踏んで設計・実装を進めたサテライトトラッカーのプロトタイプです。

最小限の軌道計算ロジック（コンソール出力）の検証からスタートし、UIの構築、パース処理、2次元描画の統合、そして時間軸制御（シミュレーション）へと肉付けしていく「アジャイルかつ手堅い開発スタイル」を採用することで、バグを最小限に抑えながらスムーズかつスピーディに高機能なアプリへと進化させることができました。

TLE（2行軌道要素形式）と地上座標を入力することで、地上から見上げた衛星の「天球配置図（Skyplot）」と、宇宙から見下ろした地球上の「可視圏（Footprint Map）」を確認でき、前後３時間分のシミュレーションも可能です。以上をマウス操作のみで完結するようUIを設計しています。

## 🚀 使い方 (クイックスタート)
このリポジトリから index.html をダウンロードします。

index.html をお好きなブラウザ（Chrome, Edge, Firefox, Safari等）で開きます。

画面内のリンクから最新のTLEや観測点座標を取得し、貼り付けるだけでシミュレーションが開始されます。

## 📊 アプリケーション・アーキテクチャ

単一のHTMLファイル（約680行）の中に、データ入力から2大スクリーンの描画にいたるまで、見通しがよく合理的な一本のデータパイプラインが通っています。

## ✨ 本アプリの優位性と特徴
他のTLE/Skyplotツールと比較しての本アプリの特徴は以下です。

 - 1ファイル完結: PythonやNode.jsのような複雑なバックエンド環境やビルド環境は一切不要で、単一のHTMLファイルをブラウザにドラッグ＆ドロップするだけで、PC・タブレットを問わず即座に動作します。

 - 数値入力は「コピペ」：衛星TLE取得のcelestrak.orgと、座標取得のGoogleMapのリンクをページ内に埋め込んでおり、いずれもコピー＆ペーストで動作します。

 - 完全オフライン動作（HTML5 Canvas）: 外部の世界地図画像や重い3Dグラフィックライブラリに依存せず、世界地図のベクター輪郭データもコードに内蔵しています。タイムスライダを動かしても描画の遅延はミリ秒単位（ほぼゼロ）という超高速動作を実現しています。ブラウザのセキュリティ制限（CORS）とも無縁です。

 - 地上（Skyplot）と宇宙（Footprint）の完全同期: 地上から見上げる視点と宇宙から見下ろす視点が、同一の数理モデルの表裏一体としてスライダの動きに滑らかに完全連動します。天体力学や衛星通信の概念を視覚的に理解するための教材としても最適です。

### v2.0.0アップデート：
衛星の軌道運動と受信環境の変化をリアルタイムに反映させるロジックを実装

 - 距離と高度の「対数スケールゲージ」:
高度400kmの低軌道（ISS）から36,000kmの準天頂軌道（みちびき）まで、大きな高度差を画面内に収めるため対数スケールによる高度ゲージを設けました。観測点から上空に伸びる線分がリアルタイムに伸縮します。

 - Range Rate（視線方向速度）に連動する相対速度(doppler)表現:
地球固定座標系（ECF）における位置・速度ベクトルから、観測者と衛星の相対速度（Range Rate）を厳密に計算しています。接近時は「青（ブルーシフト）」、遠ざかるときは「赤（レッドシフト）」へと線分の色が（速度に応じ濃淡も）変化します。仰角がマイナス（地平線下）になった瞬間に数値をグレーアウトし、現実の受信環境をに近い挙動を再現しています。(v2.0.2より高度ゲージ脇にも表示）

 - パス内限界値（ Dmin / Dmax ）の事前スキャン:
シミュレーションの対象時間帯（前後3時間）中、衛星が最も接近する限界値（Dmin：最大仰角時）と、地平線に出没する瞬間の限界値（Dmax）を常時ホールド表示します。これにより、対象衛星が描く擬似距離の把握を助けます。

 - 地球自転を考慮した「真の対地速度ベクトル」:
2D世界地図上の衛星マーカーから、地球の自転を考慮した地理的な真の進行方向（トラック角）を数学的に算出し、対地速度（km/h）の大きさに比例して伸縮するベクトル（矢印）を描画します。軌道高度による対地速度の違い、GEOやIGSOの対比など、軌道力学的な特徴を視覚化します。

### v2.8.2アップデート：
複数衛星（コンステレーション）対応と双方向ロックオンUXの統合

 - 複数衛星の自動分離とクレンジング処理：
ペーストされた複数機分のTLEから、有効なデータペア（名前＋2行）を自動で抽出します。不完全なデータ行はサイレント・オミット（自動排除）し、クラッシュを回避します。

 - 双方向コンテキスト・ロックオンUX：
Skyplot上、または世界地図上の衛星マーカーをクリックすることで、その衛星を「アクティブ衛星」として詳細データを表示します。
 
 - 厳選10色マルチカラーパレットによる視認性制御：
背景の陸地や海に同化しない高コントラストな10色の固定パレットを巡回して各衛星に割り当てます。選択中の主役衛星は不透明度100%とサイズアップ（＋レーダー状の波紋効果）で強調され、非選択の周辺衛星は半透明に抑えて描画されます。

 - 高度対数ゲージ上での全機高度プロット共存：
高度ゲージの垂直軸上に衛星をプロットすることで、軌道高度の違いを対比させます。
