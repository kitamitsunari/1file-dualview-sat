# 1file-dualview-sat
A lightweight, standalone HTML5 Canvas app that visualizes satellite orbits in both Skyplot and Footprint views via a single HTML file.

## README

本プロジェクトは Gemini Flash 3.5との対話を通じ、段階を踏んで設計・実装を進めたサテライトトラッカーのプロトタイプです。

最小限の軌道計算ロジック（コンソール出力）の検証からスタートし、UIの構築、パース処理、2次元描画の統合、そして時間軸制御（シミュレーション）へと肉付けしていく「アジャイルかつ手堅い開発スタイル」を採用することで、バグを最小限に抑えながらスムーズかつスピーディに高機能なアプリへと進化させることができました。

TLE（2行軌道要素形式）と地上座標を入力することで、地上から見上げた衛星の「天球配置図（Skyplot）」と、宇宙から見下ろした地球上の「可視圏（Footprint Map）」を確認でき、前後３時間分のシミュレーションも可能です。

## 📊 アプリケーション・アーキテクチャ

単一のHTMLファイル（約680行）の中に、データ入力から2大スクリーンの描画にいたるまで、見通しがよく合理的な一本のデータパイプラインが通っています。

```text
  ┌────────────────────────────────────────────────────────────────────────┐
  │ 1. ユーザー入力 ＆ 外部連携 (UI Layer)                                 │
  │  ├─ TLEテキストエリア ➔ 改行コード（CRLF/LF）の自動吸収                │
  │  ├─ Googleマップ座標ペースト欄 ➔ カンマ区切り文字列の自動分解・即時計算 │
  │  └─ [外部リンク導線] 📡最新TLEを取得 (CelesTrak) / 📍Googleマップ       │
  └─────────────────────────────────────────────────┼──────────────────────┘
                                                    │ (input / click イベント)
                                                    ▼
  ┌────────────────────────────────────────────────────────────────────────┐
  │ 2. 時間制御ブロック (Time Control Layer)                                │
  │  ├─ 基準時刻の保持 (baseTime = 現在時刻 / リセットボタンで同期)         │
  │  ├─ タイムスライダの現在値取得 (-180分 ～ +180分)                      │
  │  └─ 計算用対象時刻の生成 (calcTime = baseTime + スライダ値)            │
  └─────────────────────────────────────────────────┬──────────────────────┘
                                                    │
                                                    ▼
  ┌────────────────────────────────────────────────────────────────────────┐
  │ 3. 軌道・位置計算コア (Calculation Layer - satellite.js)                │
  │  ├─ twoline2satrec : TLE要素から衛星オブジェクトを復元                 │
  │  ├─ propagate       : calcTimeにおける宇宙空間の絶対位置(ECI)を計算    │
  │  ├─ gstime          : 地球の自転を考慮するための恒星時(GMST)を計算      │
  │  │                                                                     │
  │  ├─【地上視点への変換】                                                 │
  │  │   └─ eciToEcf ➔ ecfToLookAngles : 観測点から見た 方位角・仰角(AZ/EL) │
  │  │                                                                     │
  │  └─【宇宙視点への変換】                                                 │
  │      └─ eciToGeodetic              : 衛星の 直下点(緯度・経度・高度)   │
  └─────────────────────────────────────────────────┬──────────────────────┘
                                                    │
                                                    ▼
  ┌────────────────────────────────────────────────────────────────────────┐
  │ 4. 出力・二大可視化スクリーン (Presentation Layer)                      │
  │                                                                        │
  │  📊 テキスト情報の更新 (DOM操作)                                       │
  │   └─ 計算日時、AZ/EL数値、衛星位置、可視判定バッジ(見えている/隠れている) │
  │                                                                        │
  │  🎨 天球配置図スクリーン (skyplotCanvas)                                │
  │   ├─ 仰角同心円(30°/60°点線) ＆ 十字ガイド線の描画                      │
  │   └─ 極座標マッピングによる 衛星プロット(オレンジ) ＆ ラベル配置        │
  │                                                                        │
  │  🗺️ フットプリントマップ (mapCanvas)                                   │
  │   ├─ 内蔵ベクターデータによる世界地図・経緯度グリッドの高速描画        │
  │   ├─ 観測点(青) と 衛星直下点(オレンジ) のプロット                     │
  │   └─ 幾何限界角 [acos(R/(R+h))] から算出した 半透明可視圏(フットプリント)│
  └────────────────────────────────────────────────────────────────────────┘
